Restaurant Inspection Database



Above is a web application I developed for CS4400 Introduction to Database Systems. The purpose of this project is to analyze, specify, design, implement, document, and demonstrate a database management information system called the Georgia Restaurant Health Inspection System.


User may login as restaurant operator(name: chuney; password: 1234) or inspector(name:edo; password: 1234). The application will automatically guide user towards corresponding menu page based on the login information. If not registered, user can login as guest.


  • The front application is developed fully based on AngularJS. Most of the contents are generated from back end server and dynamically binded to the front end HTML elements.

  • Bootstrap is used to enhance the UI. The original UI is simple html without CSS

  • Input data is sent by $http service of AngularJS and the backend is developed using PHP & MySQL.


The detailed project description, including schema diagram and EER diagram for the database, can be found on my github.



CSS3 3d animation & AngularJS


See the Pen KdpGrM by Kaijie Huang (@hkjpotato) on CodePen.


Using CSS3 (3d rotation), I try to reproduce the animation effect of an android operating system named Smartisan OS.


  • AngularJS and SCSS are used to simplify the complexity of the code.

  • Dynamic data binding makes the outlook of every facade of the cubes changable.

  • Special concerns are given to some specific animation effect. (e.g hover effect of the color icons)

  • Details count! Parameters such as animation duration and frames are well tuned so as to provide a sense of harmony.


Through this project, I want to train myself to reproduce popular animation effects on other platforms such as IOS & Android. In the age of CSS3, I believe the touch heart moment created by entrancing animation is a key to enhance user experience and viscosity of a product.


The source materials are obtained from the official website of Smartisan's new Phone, JianGuo™. I use Photoshop to slice each facade of the cubes from the original images. Besides, I also reference the some of the codes of the original website.

Digital Menu


This is a course project of the core course of our HCI programe, CS6750 Human Computer Interaction. Working as a team with 3 other HCI students, we went through the whole process of product design, including

  • Problem identification: Task analyis & User study

  • Design alternatives: Low & high fidelity prototyping

  • Testing: Heuristic Evaluation, Cognitive Walkthrough, Usability testing

  • Design refinement and documentation, final presentation


Our project entered the CIC (Convergence Innovation Competition) and won honorable mention.


I was in charge of designing the usability testing for the product, including benchmark task, heuristic evaluation. And we conducted the user study and product testing in a professional way. Before launching the tests, a detailed IRB protocol was submitted for review. Results were well documented, discussed and integrated in the future design cycle.


The detailed report can be found on Google Drive.




Medicine Reminder


Medicine Reminder is a platform to help a medicine taker and his/her care giver to better manager the medicine taking acitivities. The platform mainly involves an IOS application for a care giver to track the medicine taking condition remotely, and a touch screen interface on the medicine container to remind the medicine taker about their pills.


Direct interviews as well as personas development were conducted to have an insight of the user scenarios. Nuanced thinking on interaction flows and interface design ensured the ease of use of the product.

Follow up: After solving a previous problem, a new problem comes. This project is like a black hole for me. A new function is post by the client: display the SSIM (what is that?!) on each camera view in real time, with a rate of 30 times/second!! Each camera view is 1280x720. To get the SSIM, I need to run image convolution of the screen shot, get the variance of those 1280x720 pixel values, and do a lot of matrix calculations. The first problem I encounter is to get the pixel values of each screen shot. The current built in function for of opentok library just return a 64base image data. To get the pixel values, I have to create an Image object with that data in javascript, and render it to an invisble canvas so as to get the RGB values. However, this is way too slow, I am now seeking to see if nodejs is faster than client side javascript, or maybe using FFT to speed up the process. Perhaps, I have to use a back end language to do multi-thread processing. Who knows? what a wonderful world!
