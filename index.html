Insider's Visual Guide to Uber

About

Uber uses an algorithm to calculate the surge price during peak times, when the demand for drivers are high in a certain area. This might have several impacts on both customers and drivers. According to a Washington Post article, these may include: a reduction in the demand for cars (less people want a ride for a higher price), creation of new supply (providing an incentive for new drivers to hit the road), and shifts in supply (drivers) to areas of higher demand.

In this project, we develope a visualization of these data to help users understand how surge multiplier works and how to optimize trip planning, especially when travelling to popular events or during peak times.

Implementation

Working with another 3 designer and developers. We make the web app for this visualization project using D3.js.

  • Provide "Overview-plus-Detail" so that the user can see the overall pattern of cost&time variation, as well as the detailed waiting time and trip duration at a specific hour
  • Link the data from various views, which is the so-called "Brushing and Linking" technique in information visualization.
    1. For bar chart, since each bar is an individual svg element, I can bind the data directly to the svg elements.
    2. For the line chart, since the line is a continuous svg element, I have to use invert scale to match the mouse position with data point.
  • Use the built-in transition function from D3 library to add animation effect to the svg element so as to enhance the user experience.

CSS3 3d Animation & AngularJS

About

I like CSS animation! This is a demo of using CSS3 (3d rotation) to reproduce the animation effect of an android operating system named Smartisan OS.

Implementation

  • AngularJS and SCSS are used to simplify the complexity of the code.
  • Dynamic data binding makes the outlook of every facade of the cubes changable.
  • Special concerns are given to some specific animation effect. (e.g hover effect of the color icons)
  • Details count! Parameters such as animation duration and frames are well tuned so as to provide a sense of harmony.

Through this project, I want to train myself to reproduce popular animation effects on other platforms such as IOS & Android. In the age of CSS3, I believe the touch heart moment created by entrancing animation is a key to enhance user experience and viscosity of a product.

The source materials are obtained from the official website of Smartisan's new Phone, JianGuo™. I use Photoshop to slice each facade of the cubes from the original images. Besides, I also reference the some of the codes of the original website.

Map Based Visualization

About

Check my Github. to see most update demo!

This is my final year research project in UX as well as my current work as a full stack developer for a map based visulization about power grid network.

A power grid network is nothing more than node and links. What special about it is that it has two levels: the high voltage level, which we call them transmission network, and low voltage level, which is called distribution network.

To show the clear hierarchy of the network based on different zoom level, I went through an interactive design process from defining design details such as color and size of elements, to providing affordance of zoomable in map components.

The detailed UX study includes:

  1. Early user + functional requirements.here
  2. A complementary user study with enter & exit surveys (5 subjects).here
  3. A detailed benchmark tasks to cover the major task flow as well as key functionalities of the product.here

The detailed report is available here.

Implementation

UX Design

After conducting the UX study, 15 design changes has been implemented to provide a better user experience. The key improvement is on the visualization, focusing on the primary feature “show different details in different levels”.

Because of that, the design should:

  1. Help user to recombine and reconnect information from different levels when user interact with it.
  2. Let user know where they are in the hierarchy.
  3. Increase affordance of exploration by providing hints.
  4. Promote clicking as because it provide direct interaction on the UI element.
  5. Reserve the interaction information from lower level when switch to higher level.
