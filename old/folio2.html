Insider's Visual Guide to Uber



About

Uber uses an algorithm to calculate the surge price during peak times, when the demand for drivers are high in a certain area. This might have several impacts on both customers and drivers. According to a Washington Post article, these may include: a reduction in the demand for cars (less people want a ride for a higher price), creation of new supply (providing an incentive for new drivers to hit the road), and shifts in supply (drivers) to areas of higher demand. We propose to present a visualization of these data to help users understand how surge multiplier works and how to optimize trip planning, especially when travelling to popular events or during peak times.


Implementation

Working with another 3 designer and developers. We make the web app for this visualization project using D3.js.

Below is one part I made for the "customer's view". The idea is to show the information the customer might be interest, such as price, waiting time, as well as total trip duration in one single window.

Monday

  • Provide "Overview-plus-Detail" so that the user can see the overall pattern of cost&time variation, as well as the detailed waiting time and trip duration at a specific hour

  • Link the data from various views, which is the so-called "Brushing and Linking" technique in information visualization.

    1. For bar chart, since each bar is an individual svg element, I can bind the data directly to the svg elements.

    2. For the line chart, since the line is a continuous svg element, I have to use invert scale to match the mouse position with data point.

  • Use the built-in transition function from D3 library to add animation effect to the svg element so as to enhance the user experience.
