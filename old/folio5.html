Follow up:

I finished a major function of this web app: the motion detection about the video stream. The motion detection is done on browser side using javascript. The major algorithim I use is the so-called SSIM (structural similarity) calculation. Basically it is used for measuring the similarity between two images by comparing the pixel values. This algorithim involves running several matrix manipulation over several successive video frames in a high rate.


Performance was the biggest challenge for me when tried to solve this problem. Afte trial and error, I used several tricks to ensure it runs on the browser without jittering effect. For example, I used typed array to store the pixel values gained from the canvas element (getImageData). As we known, each pixel has 4 values for R, G, B, and alpha, and they are stored in UInt8 type. Instead of using Uint8ClampedArray to store them, I use Int32Array with some bit calculation(e.g. bit shifting) to reduce the number of for loops.


Below is an brief illustration on how I make it (Allow the web camera if you want to test it). If a motion is detected, it will be highlighted for 2 seconds. The sensitity can be modified based on the speed of the car.


Use VanillaJS to get the web camera and stream it to the HTML5 video element.

Use drawImage function to render the video to the canvas above.

Use the getImageData to get the pixel value of the current frame. Calculate the grey scale image to reduce scale of data.

Use convolution to detect the edge, and feed the processed data to the algorithim loop so as to detect a motion.
