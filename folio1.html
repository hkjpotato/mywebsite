Restaurant Inspection Database



About

Above is a web application I developed for Database Systems Course. The purpose of this project is to analyze, specify, design, implement, document, and demonstrate a database management information system called the Georgia Restaurant Health Inspection System.

User may login as restaurant operator or inspector. The application will automatically guide user towards corresponding menu page based on the login information. If not registered, user can login as guest.

To have a quick taste of it, try the following steps

  1. login use [user: edo; password: 1234]
  2. Click the last blue button to display reports
  3. Use search criteria [year:2015; min complaint:0; max score:100]

Implementation

  • The front application is developed fully based on AngularJS. Most of the contents are generated from back end server and dynamically binded to the front end HTML elements.

  • Bootstrap is used to enhance the UI. The original UI is simple html without CSS

  • Input data is sent by $http service of AngularJS and the backend is developed using PHP & MySQL.

The detailed project description, including schema diagram and EER diagram for the database, can be found on my github.
